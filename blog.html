Question-1: What are the different ways to select an element in the DOM?

Ways to Select an Element in the DOM- getElementById("id") : Retrieves a single element using its ID. getElementsByClassName("class") : Selects multiple elements by class name, returning an HTMLCollection. getElementsByTagName("tag") : Retrieves multiple elements by tag name, also returning an HTMLCollection. querySelector("selector") : Selects the first element that matches a given CSS selector. querySelectorAll("selector") : Retrieves all elements matching a CSS selector, returning a NodeList. For a single element, use getElementById() or querySelector(). For multiple elements, use getElementsByClassName(), getElementsByTagName(), or querySelectorAll().

Question-2: What is the difference between innerHTML, innerText, and textContent?

The key difference between `innerHTML`, `innerText`, and `textContent` is how they retrieve and manage content within an element. The `innerHTML` property allows getting or setting the HTML content inside an element, meaning it includes both text and any embedded HTML tags. This makes it useful for handling formatted content but can be a security risk when dealing with user input. In contrast, `innerText` retrieves or updates only the **visible text**, ignoring hidden elements and stripping out any HTML tags. This makes it a faster and safer option for basic text manipulation. Meanwhile, `textContent` also retrieves or sets text within an element, but unlike `innerText`, it includes hidden text while still treating all content as plain text without rendering HTML. Because of its efficiency and security benefits, `textContent` is often preferred when working with raw text. In summary, `innerHTML` is best for handling structured HTML content, `innerText` is ideal for working with only visible text, and `textContent` is suitable when extracting all text without any HTML formatting.

Question-3: What is event delegation in the DOM?

Event delegation is a JavaScript technique where a parent element manages events for its child elements, eliminating the need to attach event listeners to each child separately.

Question-4: What is event bubbling in the DOM?

Event bubbling in JavaScript is a process where an event originates from the deepest (innermost) element and then propagates upward to its parent elements. If a button is inside a div, and both have click event listeners. Clicking the button first triggers its event.The event then moves up to the div (its parent).It continues bubbling up to the body and even the HTML document.
